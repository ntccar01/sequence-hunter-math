# 互動式教學模式

## 定位

本課程的互動式網頁不是單純答題網站，而是「RPG 遊戲化 + 紙本學習單 + 即時學習分析」的教學平台。

## 基本架構

```text
學生端網頁
→ 題庫資料
→ 學生送出答案
→ Google Apps Script API
→ Google Sheet 學習紀錄
→ 教師戰情室
```

## 每個小單元的設計流程

1. 擷取課本核心知識點。
2. 整理課本例題與類題。
3. 判斷例題是否適合轉化為 RPG 任務。
4. 將適合的例題設計成關卡。
5. 將不適合遊戲化的內容放入紙本學習單。
6. 類題設計為練習題、挑戰題或補救題。
7. 為每題設計公式提示、代號說明、思考引導與學習單提示。
8. 建立教師端可觀察的學習資料。

## 題庫欄位建議

```text

```

## 關卡類型建議

- 暖身題：降低進入門檻，建立學生願意作答的第一步。
- 例題任務：承接課本例題，轉化為故事或任務情境。
- 關鍵步驟題：不只問答案，也問學生下一步該做什麼。
- 類題練習：讓學生在紙本計算後於網頁送出答案。
- Boss 題：整合該小單元概念，適合作為課末挑戰。

## 低成就學生設計原則

- 答錯不懲罰，改成再挑戰。
- 提示分層出現，不一次給完。
- 每次只要求學生完成一個明確任務。
- 題目數量先少後多。
- 教師端優先觀察「未作答」、「多次錯誤」、「卡在同一題」。
